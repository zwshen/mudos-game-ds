我來講講我當初是怎麼起家的好了? :p 
 
大概三年多前, 我那時還是用 AMD 386 DX-40 with 2M ram.. 
 
那時候剛知道 Mud 這個東西. 
跑去玩幾天後, 想說這東西是怎麼弄出來的呀.. 
 
所以就貼了篇 post 在這邊問. 
 
我那時問的是: "請問..要在哪裡才能找到架 Mud 相關的東西呀?" 
等了一兩天, 有人告訴我 ccca 所屬的 ftp site 下有.. 
 
可是, 他沒告訴我在哪個目錄.. :p 
我就是從那一次學會怎麼樣在大ftp 裡翻東西的..:p 
 
找到後, 什麼是 .tar.gz, .tgz, .tar.Z 啊..@_@ 
問了朋友後知道, 那個是 unix 下常用的壓縮跟打包用的格式 (tar, gzip, compress) 
 
好吧, 乖乖的買本 unix command 的書回家看.. (這年頭的人幸福多了, 現在的書 
水準跟量都比三年多前進步多了..) 
 
可是, 我的帳號那時沒這麼大的quota..怎麼辦呢? 
看了看解開後的 Makefile, 啥.. CC = gcc 
 
gcc gcc, 嗯..unix command書裡提到, 用 man 可以查詢命令.. 
所以, man gcc .. 
 
啪啦啪啦的捲過去一堆東西.. 老實說我也是沒全看完 :p 
(看英文其實我個人的要訣是, 找一些你要的keyword..從那邊看起 :p) 
 
看著看著, 啥?! 看到 386 ... 我就想說, 這東西該不會有 pc 版吧 :D~ 
 
man gcc 最上面很明白的寫著: 
GCC(1)                      GNU Tools                      GCC(1) 
                            ^^^ 
NAME 
       gcc, g++ - GNU project C and C++ Compiler (v2.7) 
                  ^^^^ 
SYNOPSIS 
       gcc [ option | filename ]... 
       g++ [ option | filename ]... 
 
WARNING 
       The  information  in  this man page is an extract from the 
       full documentation of the GNU C compiler, and  is  limited 
                                 ^^^ 
       to the meaning of the options. 
 
GNU? 這是哪家公司呀.. 我那時這樣想. (我那時unix也剛開始玩) 
我也知道 Unix 下產的東西都很貴, 再加上 man gcc 看到, gcc 似乎 support 
不少平台.. 這樣的東西要買會不會很貴啊? :~ 
 
傻傻的跑去拿了一些軟體列表, 找了半天.. 
看到了 borland c++, M$ c, M$ VC, watcom c ..一堆 c compiler.. 
沒看到 GNU 呀..:~ 
 
跑去 software board, programming board, Unix/bsd/linux board 翻 post/精華區 
這才知道, gcc 是不用錢的 :D~~ 
 
大概算我幸運吧, 剛好在 programming board看到 gcc for dos 是 djgpp 
 
透過archie 一找.. (那年頭 www 可不流行呀..:p, 用archie還沒yahoo一類的 
網站方便..) 天啊, djgpp 這麼大 :~ (那時不知道只要抓幾個必要的就好了..:p) 
用 modem 怎麼傳啊 .. 
 
幸而..剛好當期的 hope net 光碟月刊的cd 裡面有收錄 djgpp 1. .(還真巧 :p) 
 
我就這樣子用 2M ram 開始在 386 上奮鬥~~|) 
 
曾經 port 過 merc 2.2, envy 1.0, MudOS忘了哪一版..到 DOS 下 (use djgpp) 
改過中文的問題. 
 
改上面的東西, Makefile要會改, 要懂 c 語言.. 呃, 不是會寫作業就夠了.. 
 
c 語言我想, 我大概很難給你任何學習上的經驗 (自己到今天還是搞不懂自己在 
寫什麼東西..:~) 
 
Makefile 對不少人來說可能要花點心思了解一下 makefile rule 
我比較佔便宜的是因為以前用 turbo c, M$ c 5 的時候, M$ c 5 是一定 
得寫 makefile.. turbo c 則是基於 exec file size 的問題, 我最後一定 
會以 command line的 tcc 去編.. 當然, project 一定是寫 makeifle 方便 :p 
 
merc/diku 系的 mud..driver code裡會用到的主要是 linked list.. 
所以 linked list一定要熟. 
 
hash, 函式指標, 字串的 parse, 記憶體管理, 網路通訊 (socket program).. 
大量巨集的使用, 還有熟練的 debug 經驗.. (merc 2.2 with mob-program 
有一堆小地方..感覺上是無心之過, 不過留下來的麻煩很大) 
 
lpmud..我研究的不多, 不敢說什麼.. 
 
mudlib..感覺上是, 其實精神上比較像java..:p 雖然語法很像 c.. 
還有, driver 本身提供的 efun, 變數等.. 
以及 driver 的定義..要弄懂 
 
我自己的感覺是, 單純抓 MudOS 回來, doc 的編排法似乎不太適合初學者 
你一定要先弄等他的精神, 進而知道有哪些 '物件 及 '函式' 是必要的 
和必須被overwrite的.. 
 
而不是給你一份 '指引手冊', 告訴你該怎麼起步.. 
同時, MudOS 的 docs 常會漏掉一些東西.. :p 
 
那怎麼辦? ..看source code最快.. 
不然後面會衍生出來的問題更多.. 
 
舉個例, in config file: 
# the file which defines the master object 
master file : /kernel/daemon/master 
 
# the file where all global simulated efuns are defined. 
simulated efun file : /kernel/simul_efun/simul_efun 
 
為什麼要這兩個設定? 分別又是作什麼用的? 
 
Master obj 的用途何在? 什麼是 efuns? 什麼是 simulated efuns? 
efuns 又是作什麼用的? 
 
Master obj 裡又要定義什麼? 
 
所以, 很多時候 post 就丟一句: 我要架mud, 誰可以幫我? , 怎麼架 mud? 
要回答的問題之多, 我想, 就我的立場來說, 除了給人的感覺很沒誠意外.. 
這個問題被後包含的問題之多, 一來不是一兩篇post就能解釋得清楚, 
二來, 實在也不知道發問的人到底是對這浩瀚的問題裡, 所不知的為何.. 
 
怎麼說問題太多? 
就我上面的起步來講.. 
1. 你要會有自己找資源的能力 
2. 你要會寫 c, 要熟悉基本的資料結構/演算法 
3. 對於你工作的平台的特性要熟悉. (雖說 c 是號稱可以跨平台, 不過事實上在 
   踫到某方面的bug時, 解決問題的方法還是對該平台有相依性) 
4. 對於該編譯器熟悉, 並善用其提供的周邊工具. (gnu c的大概也要對unix commands 
   , shell script..等熟, borland c++, c++ builder, M$ vc users要面對的麻煩更 
   多.. 說真的.) 
5. driver 本身運作流程要熟悉.. 
   如果你清楚, 那上面問 Master obj, simulated efuns 的問題就不是問題了.. 
   diku/merc 系的 mud 不清楚的話..嘿嘿..|) 
6. 詳讀所附的文件.. 很多問題其實裡面就有基本指引方向了.. 
 
上面這些..暫時想到的. 好像沒哪一點可以速成.. (我很笨的) 
大概看個幾萬行原始碼的日子是跑不掉的.. 
 
這也難怪有時候..看到鳥問題會口氣不好.. :P 
 
現在找資源已經方便多了..(如果您還是找不到, bbs.ntu.edu.tw mud board 
的精華區有一些關於架mud的resource list) 
問題是怎麼找 :) 
 
InterNet是很大的, 大到你窮一生之力都玩不完.. 
要怎樣從這麼多資源 (垃圾? :p) 中找到你要的寶貝.. 要自己發展一套法子 
呀.. :) 
 
我笨笨的摸了這麼多年, 用的是最沒效率的法子在工作.. 
相信各各比我聰明得多了, 一定能事半功倍.. 
給各位想成為大神的新手一點借鏡 :)
