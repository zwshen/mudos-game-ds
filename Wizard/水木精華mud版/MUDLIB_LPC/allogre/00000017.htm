=========================================== 
===============  多邊形渲染  ============== 
=========================================== 
注意: 這些函數不支持真彩顯示模式! 
void polygon3d(BITMAP *bmp, int type, BITMAP *texture, int vc, V3D *vtx[]); 
void polygon3d_f(BITMAP *bmp, int type, BITMAP *texture, int vc, V3D_f *vtx[]); 
   向指定位圖畫 3D 多邊形, 使用指定的渲染模式.和常規的 polygon() 
   函數不同, 這些例程不支持凹多邊形或自交叉多邊形, 並且它們不能向 
   mode-X 屏幕位圖上畫 (如果你想寫 mode-X 下的 3D 代碼, 向內存位 
   圖作畫然後 blit 到屏幕). 貼圖的寬度和高度必須是 2 的整數次方, 
   但可以不同, 比如: 可以是 64x16 的貼圖, 但不能是 17x3 的. 頂點 
   計數參數  (vc) 應當跟在一個包含有合適數量的頂點結構指針的數組 
   之後: polygon3d() 使用定點 V3D 結構指針, 而 polygon3d_f() 使用 
   浮點 V3D_f 結構指針. 
 
   下面是它們的定義: 
   typedef struct V3D 
   { 
      fixed x, y, z;       - 位置 
      fixed u, v;          - 貼圖坐標 
      int c;               - 顏色 
   } V3D; 
 
   typedef struct V3D_f 
   { 
      float x, y, z;       - 位置 
      float u, v;          - 貼圖坐標 
      int c;               - 顏色 
   } V3D_f; 
   頂點數據的使用方法依賴於渲染模式: 
 
   x 和 y 值指定了頂點在 2d 屏幕坐標上的位置. 
   z 值僅僅在用在作貼圖透視糾正時,它指定了點在 3d 世界坐標中 
   的深度. 
   u 和 v 坐標僅在做貼圖時使用, 它們指定了點在貼圖中的位置, 
   比如 0,0  映射頂點到貼圖的左上角, 如果貼圖是 32x32 的, 設 
   置 u=32 和 v=16 則映射頂點到右邊界的中點. u/v 坐標對貼圖的 
   邊界是翻捲的, 因此在 32x32 的貼圖裡, u=v=32 和  u=v=0 相同. 
   這可以讓貼圖在一個多邊形裡被拼接多次. 
 
   c 值指定了頂點的顏色, 根據不同的渲染模式對此有不同的解釋. 
 
   type 參數指定了多邊形的渲染模式, 它只能是下列值之一: 
 
   POLYTYPE_FLAT: 
      簡單的平展作過陰暗處理的多邊形, 顏色取自第一個頂點的顏色. 
 
   POLYTYPE_GCOL: 
      一個簡單顏色過度的作過陰暗處理的多邊形. 每個頂點的顏色取 
      自 c 值, 並且添入多邊形. 它非常的快, 但是你的調色板必須包 
      含有平滑過渡的色彩. 
 
   POLYTYPE_GRGB: 
      一色彩過渡的作過陰影處理的多邊形, 以 RGB 色組染色而不是單 
      色調, 它使用全局 reb_map 表來轉換結果到 8 位調色板顏色.  
      必須在你設置了 RGB 映射表後才能使用! 這個模式比 POLYTYPE_GCOL 
      要慢的多, 但是不需要調色板包含所有你將使用的過渡色彩. 每 
      個頂點的顏色都取自 c 值, 這個 c 值是 24 位 RGB 顏色組形式 
      (0xFF0000 是紅色, 0x00FF00 是綠色, 0x0000FF 是蘭色). 
 
   POLYTYPE_ATEX: 
      仿射(affine)貼圖多邊形. 它用簡單的 2d 線形方式拉伸貼圖 
      添入多邊形, 這很快但是沒有數學矯正. 當多邊形相當小或平面 
      正對攝像機時看起來不錯, 但是因為沒有經透視前縮處理, 它導 
      致了奇怪的翻捲加工. 要明白我的意思, 運行 test.exe 看看 
      polygon3d() 測試時當你放大貼近立方體時的效果. 
 
   POLYTYPE_PTEX: 
      透視矯正貼圖多邊形. 它使用頂點結構中的 z 值用在 u/v 坐標中, 
      因此貼圖能在任何角度被正確顯示.因為它在貼圖內循環中使用了除 
      法, 這個模式比 POLYTYPE_ATEX 慢很多, 並且它使用了浮點指針因 
      此在奔騰機以下會很慢(即使有 FPU, 486 也不能像奔騰那樣將浮點 
      指針和其它整數運算間作除法). 
 
   POLYTYPE_ATEX_MASK: 
   POLYTYPE_PTEX_MASK: 
      與 POLYTYPE_ATEX 和 POLYTYPE_PTEX 類似, 但是貼圖中的 0 點被 
      忽略, 這樣允許了貼圖中的部分透明. 
 
   POLYTYPE_ATEX_LIT: 
   POLYTYPE_PTEX_LIT: 
      與 POLYTYPE_ATEX 和 POLYTYPE_PTEX 類似, 但是全局 color_map 
      表被用來混合貼圖上取自頂點結構中 c 值的光線級別.這必須在你 
      設置了色彩映射表後才能使用! 這些模式不能被用在大於 256x256 
      的貼圖. 
 
void triangle3d(BITMAP *bmp, int type, BITMAP *tex, V3D *v1, *v2, *v3); 
void triangle3d_f(BITMAP *bmp, int type, BITMAP *tex, V3D_f *v1, *v2, *v3); 
   畫一個 3d 三角形, 使用頂點或浮點的頂點結構. 這些相當於調用 
      polygon3d(bmp, type, tex, 3, v1, v2, v3); 
   或 
      polygon3d_f(bmp, type, tex, 3, v1, v2, v3); 
 
void quad3d(BITMAP *bmp, int type, BITMAP *tex, V3D *v1, *v2, *v3, *v4); 
void quad3d_f(BITMAP *bmp, int type, BITMAP *tex, V3D_f *v1, *v2, *v3, *v4); 
   畫一個 3d 四邊形, 使用頂點或浮點的頂點結構.這些相當於調用 
      polygon3d(bmp, type, tex, 4, v1, v2, v3, v4); 
   或 
      polygon3d_f(bmp, type, tex, 4, v1, v2, v3, v4);
