=========================================== 
============     鍵盤例程      ============ 
=========================================== 
如果你想檢測同時按下的多個按鍵, BIOS 
鍵盤例程就沒什麼用了. Allegro 能加載一個鍵盤處理程序的替代品, 
它提供了緩衝輸入和標記集合兩種方式來儲存每個鍵的狀態. 
注意,由於 PC 鍵盤本身的設計,它不可能正確的檢測到每種按鍵的組合, 
同時按下兩到三個鍵將工作的很好, 
但是如果你同時按下更多的鍵, 
多餘的部分將被忽略 (準確的說得到的鍵的組合隨著每一個鍵盤而不同). 
 
int install_keyboard(); 
   加載 Allegro 鍵盤中斷處理程序.你必須在使用 
   任何鍵盤輸入例程前調用它.一旦你設置了 
   Allegro 處理程序,就不再需要使用 DOS/BIOS 調用或者 C 庫函數 
   來控制鍵盤了. 
void remove_keyboard(); 
   卸載鍵盤處理程序,將控制權交回 BIOS. 正常的情況你不必 
   自己調用它,因為 allegro_exit() 將為你做這些. 
 
void install_keyboard_hooks(int (*keypressed)(), int (*readkey)()); 
   只有在你 *不需要* 使用鍵盤處理程序的其它部分時才使用這個函數. 
   它應該在 install_keyboard() 的地方被調用, 
   讓你提供一個檢測和讀取鍵盤的回調例程, 
   這個例程將被 keypressed() 和 readkey() 調用. 
   如果你想使用 Allegro 的 GUI 代碼和自己配制的鍵盤處理程序時這就很有用, 
   因為它提供了用你自己代碼為 GUI 控制鍵盤輸入的一個方案. 
   如果你想使用 BIOS 鍵盤例程, 
   libc 函數 _bios_keybrd(_KEYBRD_READ) 將以正確形式返回鍵擊. 
 
extern volatile char key[128]; 
   以掃描碼的順序描述每個按鍵狀態的標誌數組. 
   掃描碼以一串 KEY_* 形式的常量定義在 allegro.h 裡. 
   比如,你可以這樣寫: 
      if (key[KEY_SPACE]) 
         printf("Space is pressed\n"); 
   每個值都可以以在某些位上包含標誌 
   KB_NORMAL 和 KB_EXTENDED, 這使得你可以檢測出具有相同掃描碼的鍵 
   中具體按下的是哪一個.例如, 
   你可以這樣寫: 
      if (key[KEY_ENTER] & KB_NORMAL) 
         printf("Enter (next to the right shift key) is pressed\n"); 
      if (key[KEY_ENTER] & KB_EXTENDED) 
         printf("Enter (on the numeric keypad) is pressed\n"); 
extern volatile int key_shifts; 
   這個位屏蔽變量包含了當前的 shift/ctrl/alt 狀態, 特別的 
   Windows 鍵,和 accent escape 字符. 可以含有以下標誌: 
 
      KB_SHIFT_FLAG 
      KB_CTRL_FLAG 
      KB_ALT_FLAG 
      KB_LWIN_FLAG 
      KB_RWIN_FLAG 
      KB_MENU_FLAG 
      KB_SCROLOCK_FLAG 
      KB_NUMLOCK_FLAG 
      KB_CAPSLOCK_FLAG 
      KB_INALTSEQ_FLAG 
      KB_ACCENT1_FLAG 
      KB_ACCENT1_S_FLAG 
      KB_ACCENT2_FLAG 
      KB_ACCENT2_S_FLAG 
int keypressed(); 
   如果在輸入緩衝中有鍵擊則返回 TRUE. 
   相當於 libc kbhit() 函數. 
int readkey(); 
   返回鍵盤緩衝裡的下一個字符.如果緩衝是空的, 
   則等待下一次鍵擊.返回值的低字節 
   是鍵的 ASCII 碼,高字節為掃描碼. 
   掃描碼無論在何種 shift, ctrl 和 alt 狀態下都保持一致. 
   ASCII 碼以正常的方式被 shift 和 ctrl 的狀態影響. 
   (shift 改變大小寫, ctrl+字母則給出字母在字母表中的位置, 
   即 ctrl+A = 1, ctrl+B = 2, 等等). 按下 alt+key 只返回掃描碼, 
   而低字節為零. 比如: 
      if ((readkey() & 0xff) == 'd')         // 檢查 ASCII 碼 
         printf("You pressed 'd'\n"); 
      if ((readkey() >> 8) == KEY_SPACE)     // 檢查掃描碼 
         printf("You pressed Space\n"); 
      if ((readkey() & 0xff) == 3)           // ctrl+字母 
         printf("You pressed Control+C\n"); 
      if (readkey() == (KEY_X << 8))         // alt+字母 
         printf("You pressed Alt+X\n"); 
void simulate_keypress(int key); 
   向鍵盤緩衝塞入一個鍵,就像用戶親自敲過一樣. 
   參數和 readkey() 的返回值的格式相同. 
extern int (*keyboard_callback)(int key); 
   如果被設置, 鍵盤處理程序在反應每次鍵擊時都調用這個函數. 
   將即將加入輸入緩衝的值傳入這個函數, 
   然後你可以不加改變的將這個值返回,也可以 
   返回零使得按鍵被忽略,或者返回一個修改的值來改變 
   readkey() 將返回的值. 
void set_leds(int leds); 
   設置鍵盤 LED 指示器的狀態. 參數是一個 
   可以包含 KB_SCROLOCK_FLAG, KB_NUMLOCK_FLAG, 和 KB_CAPSLOCK_FLAG 這些值 
   的位掩值 (bitmask), 或者傳入 -1 來恢復缺省狀態. 
void clear_keybuf(); 
   清空鍵盤緩衝. 
extern int three_finger_flag; 
   Allegro 鍵盤處理程序提供了一個 '緊急出口' 的序列,可以用來 
   殺掉你的程序. 如果你正在 DOS 下運行, 這是 
   三個手指的活, ctrl+alt+del. 所有的多任務 OS 都會 
   在 Allegro 處理之前捕捉這個組合, 所以 
   你可以使用改變的鍵組 ctrl+alt+end. 如果你想在 
   你的程序的發行版中關掉這個行為, 將這個標誌設為 FALSE. 
 
extern int key_led_flag; 
   缺省狀態下, capslock, numlock, 和 scroll-lock 鍵在它們被觸動時 
   聯繫著鍵盤 LED 指示器. 如果你在你的遊戲中使用了這些做輸入鍵 
   (比如. capslock 為開火鍵) 這個就不那麼 
   稱心如意了, 因此你可以清除這個標誌來阻止 LED 的狀態被改變.
