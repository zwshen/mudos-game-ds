================   設置例程       ============== 
================================================ 
Allegro 中可變的部分, 例如聲音例程和 
load_joystick_data() 函數, 需要一些設置信息. 這些 
數據被儲存在一個文本文件的 '變量=值' 這樣的行中, 
注解被寫在以 '#' 開頭的行,知道行末. 
設置文件可以被任意的分成幾個部分(section), 
每個部分以 "[這部分的名字]" 這樣一行開始. 每部分有唯一的名字空間, 
這樣是避免名字的衝突,但是只要不在同一部分, 
每個變量都可以同時出現在所有的部分了. 
缺省時設置數據是從一個叫 allegro.cfg 或 sound.cfg 的文件裡讀入的, 
這個文件將在和程序所處的目錄下尋找, 
或者放在 ALLEGRO 環境變量指向的目錄裡. 
如果你不喜歡這中方式,你可以鎮定任何一個你喜歡的文件名,或者 
使用你自己程序規定的一塊二進制設置數據 
(比如可以從數據文件裡載入). 
你可以在設置文件裡儲存你喜歡的任何配置信息, 
就像 Allegro 使用的基本變量那樣 (見後面). 
void set_config_file(char *filename); 
   設置設置文件名,以被附隨的所有設置函數使用. 
   如果你沒有調用這個函數, Allegro 將使用缺省的 allegro.cfg 
   這個文件,首先在和執行程序所在的目錄下尋找, 
   然後在 ALLEGRO 環境變量指向的目錄下尋找. 
void set_config_data(char *data, int length); 
   指定一個已經被載入內存的數據塊來被所有的附隨設置函數使用, 
   (可以是一個你自己的複雜數據類型,或者 
   從 grabber 數據文件裡載入的數據). 這個例程 
   將信息作了備份,所以你在調用它後可以安全的釋放掉數據佔的內存. 
 
void override_config_file(char *filename); 
   指定一個包含了優先設置的文件名,這些設置 
   將被用在主設置文件的參數附加信息上,當一個 
   變量在兩個文件中均被提及,則這個版本中的變量將被優先採用. 
   這些可以被應用程序的程序員在他們的代碼中優先使用一些設置, 
   然後依舊將主設置文件留給用戶配置. 
   例如, 你可以指定一個 
   特別的樣本頻率和 IBK 樂器(instrument)文件,但是用戶還是 
   可以用 sound.cfg 或 allegro.cfg 文件來指定端口設置和 
   IRQ 數值. 
void override_config_data(char *data, int length); 
   override_config_file() 的使用內存數據塊版本. 
 
void push_config_state(); 
   將當前的設置狀態(文件名,變量值,等)壓入一個內部棧 , 
   使你可以選擇其它的設置, 
   然後可以調用 pop_config_state() 來恢復當前的設置. 
   這個函數是被設計成其它庫函數內部使用的, 
   比如當你為 save_joystick_data() 指定一個設置文件名, 
   它在切換到你指定的文件前就將設置狀態壓棧. 
 
void pop_config_state(); 
   將前面用 push_config_state() 儲存的設置狀態彈出, 
   用它來替換掉當前的設置. 
char *get_config_string(char *section, char *name, char *def); 
   從當前的設置文件裡得到一個字符串.如果變量名 
   沒有被找到,則返回缺省值. section 名可以被設為 NULL 
   來從任意的部分中尋找值,或者用來 
   控制你感興趣的參數集合 (例如 sound 或 joystick). 
 
int get_config_int(char *section, char *name, int def); 
   從當前的設置文件裡讀一個整數值. 
   見 get_config_string() 的注解. 
int get_config_hex(char *section, char *name, int def); 
   以十六進制讀取當前設置文件裡的一個整數值. 
   見 get_config_string() 的注解. 
float get_config_float(char *section, char *name, float def); 
   從當前設置文件裡讀取一個浮點值. 
   見 get_config_string() 的注解. 
char **get_config_argv(char *section, char *name, int *argc); 
   從當前設置文件裡讀取一個標記列表 (用空格分開的單詞), 
   返回一個 argv 類型的列表, 將 argc 設置為標記的個數 
   (不像 argc.argv, 這個表是從 0 開始的). 
   如果變量沒有提供,返回 NULL 並將 argc 設為 0. 
   標記列表被儲存在一個臨時緩衝區,數據將被下次調用 
   get_config_argv() 摧毀, 所以數據不會長期保留. 
void set_config_string(char *section, char *name, char *val); 
   向當前的設置文件寫一個字符串數據,覆蓋掉原來的任何 
   可能存在的數據,如果 val 是 NULL 的話則移去這個變量. 
   section 名字可以為空,以用來將數據寫在文件的根部分, 
   或者用來控制有這個變量的 section. 
   這個被改變的文件將放在內存緩衝, 直到你調用 allegro_exit() 
   才真正寫入磁盤. 注意,用這個方法你只能寫文件, 
   所以如果當前設置源是被 set_config_data() 指定的話,函數將不起效果. 
   (只對 set_config_file() 指定) 
void set_config_int(char *section, char *name, int val); 
   向當前設置文件寫一個整數值. 
   見 set_config_string() 的注解. 
void set_config_hex(char *section, char *name, int val); 
   以十六進制格式向當前的設置文件寫一個整數. 
   見 set_config_string() 的注解. 
void set_config_float(char *section, char *name, float val); 
   向當前設置文件寫一個浮點數值. 
   見 set_config_string() 的注解. 
Allegro 在設置文件了使用了以下這些基本變量: 
gfx_card = x 
   指定當程序以 GFX_AUTODETECT 檢測時使用的圖形驅動器. 
   可以用額外行 "gfx_card2 = x", "gfx_card3 = x" 等來給出多個驅動器的建議. 
   驅動器的 ID 必須是下面值中的一個: 
 
      1  = VGA mode 13h          2  = Mode-X 
      3  = VESA 1.x              4  = VBE 2.0 (banked) 
      5  = VBE 2.0 (linear)      6  = VBE/AF 
      7  = Xtended mode          8  = ATI 18800/28800 
      9  = ATI mach64            10 = Cirrus 64xx 
      11 = Cirrus 54xx           12 = Paradise 
      13 = S3                    14 = Trident 
      15 = Tseng ET3000          16 = Tseng ET4000 
      17 = Video-7 
keyboard = x 
   指定一個鍵盤佈局來使用. 參數是一個 
   用鍵盤設置工具生成的鍵盤映射文件名,這可以是 
   一個合格的含路徑文件名或是像 "us" 或 "uk" 這樣的基本名. 
   之後, Allegro 將首先查找這個指定的文件 (比如 "uk.cfg"), 
   然後查找 keyboard.dat 文件裡的同名對像 (比如 "UK_CFG"). 
   這個配置文件或 keyboard.dat 文件可以放在程序的同一目錄, 
   或者是 ALLEGRO 環境變量所指向的目錄. 
   在 keyboard.dat 文件裡看哪一種鍵盤映射可用: 
   這時有以下選擇: 
 
      BE       - 比利時 
      DE       - 德國 
      DVORAK   - 丹麥 
      ES       - 西班牙 
      FI       - 芬蘭 
      FR       - 法國 
      PT       - 葡萄牙 
      UK       - 英國 
      US       - 美國 
mouse = x 
   鼠標類型 (Microsodt, Logitech, 或 NT). 任何一種方式都 
   可以運行在任何系統上, 但是 Microsoft 代碼可能更可信點. 
   一些 Logitech 驅動器有個小 bug,使鼠標已 8 倍速移動, 
   所以,如果你踫到這種問題時就要使用 Logitech 的設置. 
 
[sound] (聲音) 
   這部分包含了聲音設置信息,使用了這些變量: 
digi_card = x 
   設置播放採樣聲音的驅動器, x 可以是這些值: 
      0 = 無                     1 = SB (自動檢測) 
      2 = SB 1.0                 3 = SB 1.5 
      4 = SB 2.0                 5 = SB Pro 
      6 = SB16                   7 = GUS (未完成) 
midi_card = x 
   設置播放 MIDI 音樂的驅動器, x 可以是這些值: 
      0 = 無                     1 = Adlib (自動檢測 OPL 版本) 
      2 = OPL2                   3 = 雙(Dual) OPL2 (SB Pro-1) 
      4 = OPL3                   5 = SB MIDI 界面 
      6 = MPU-401                7 = GUS (未完成) 
      8 = DIGMID                 9 = AWE32 
digi_voices = x 
   指定為數字聲音驅動器保留的最少聲音數量. 
   多少可能依賴於驅動器. 
midi_voices = x 
   指定為 MIDI 聲音驅動器保留的最少聲音數量. 
   多少依賴於驅動器. 
flip_pan = x 
   將其設為 0 或 1 來反轉採樣聲音的左/右聲道, 
   這個功能有時是必要的,因為有些 SB 卡 (包括我的 :-) 的立體聲的方向錯了. 
 
sb_port = x 
   設置 SB 的端口地址 (一般是 220). 
sb_dma = x 
   設置 SB 的 DMA 通道 (一般是 1). 
sb_irq = x 
   設置 SB 的 IRQ (一般是 7). 
sb_freq = x 
   設置採樣聲音的頻率, 缺省值為 16129. 可能的值有: 
      11906 - 任何 SB 都可用 
      16129 - 任何 SB 都可用 
      22727 - 工作在 SB 2.0 及以上版本 
      45454 - 只能工作在 SB 2.0 或 SB16 上 (不能用在 stereo SB Pro) 
fm_port = x 
   設置 OPL synth 的端口地址 (一般是 388). 
mpu_port = x 
   設置 MPU-401 MIDI 界面的端口地址 (一般是 330). 
digi_volume = x 
   設置數字採樣聲音播放時的音量, 可以從 0 到 255. 
midi_volume = x 
   設置 midi 音樂播放時的音量, 可以從 0 到 255. 
ibk_file = x 
   指定 .IBK 文件的名字, 它將取代 
   基本 Adlib patch. 
ibk_drum_file = x 
   指定 .IBK 文件的名字,它將取代 
   基本 Adlib percussion patches. 
patches = x 
   指定從何處尋找 DIGMID 驅動器的採樣設置. 
   這可以是一個 Gravis 格式的包含一些 .pat 文件和一個 default.cfg 索引的目錄, 
   或是用 pat2dat 工具建立的 Allegro 數據文件. 
   如果這個變量沒有被設置, Allegro 將查找 default.cfg 和 patches.dat 文件, 
   它們可以放在和程序相同的目錄下,  
   ALLEGRO 環境變量指向的目錄, 
   ULTRASND 環境變量指向的基本 GUS 目錄. 
[midimap] 
   如果你在使用 SB MIDI 輸出或 一個有外部合成器但不是 
   General MIDI 兼容的 MPU-401 驅動器, 你可以使用設置文件中的 midimap 
   部分去指定一個映射表(mapping table)補丁來 
   將 GM 補丁項轉換成任何其它的組合(bank), 
   程序變更消息(program change messages)將在你的 synth(合成)上選擇合適的聲音. 
   這真的有點自我放縱. 我有一塊 Yamaha TG500, 它有一些很棒的聲音 
   但是沒有 GM 補丁設置, 我剛才使它莫名其妙的工作起來... 
   這個部分由一組下面這樣格式的行組成: 
p<n> = bank0 bank1 prog pitch 
   這裡, n 是 GM 程序改變數字 (1-128), bank0 和 
   bank1 是要送到你的 synth(合成) 的兩個變更消息庫(控制器 #0 和 #32), 
   prog 是要送到你的 synth (合成) 的程序變更消息, 
   pitch 是用那個聲音播放時需要移動的半音的個數. 
   將組合(bank)變更數設為 -1 將 
   防止它們被送出. 
   舉個例子,下面這行: 
      p36 = 0 34 9 12 
   指定當 GM 的 36 號程序(program) (發生一個 fretless bass) 被選擇, 
   Allegro 將發送參數為 0 的組合(bank)變更消息 #0, 
   參數為 34 的組合(bank)變更消息 #32, 
   參數為 9 的程序(program) 變更消息, 然後將所有的聲音 
   提高八度. 
[joystick] (遊戲杆) 
   這個部分包含了一組由 save_joystick_data() 和 
   load_joystick_data() 使用的變量.
