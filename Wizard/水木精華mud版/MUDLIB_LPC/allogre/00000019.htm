請先閱讀 readme.txt 裡的一般介紹,版權條款,和 有關如何編譯 Allegro 的信息.   
 
所有的 Allegro 函數, 變量, 數據結構都定義在 allegro.h 裡. 你必須在你的程序裡包含  
(include)這個文件, 並且連接(link) liballeg.a 為了做到這些,你必須:   
 
- 將下面這行寫到所有使用了 Allegro 的 C 或 C++ 文件的開始:   
 
 
#include <allegro.h>   
 
- 如果你在命令行下或利用 makefile 編譯程序, 將 '-lalleg' 加到 gcc 命令行的末尾,  
比如:   
 
gcc foo.c -o foo.exe -lalleg   
 
- 如果你在使用 Rhide, 選擇 Options/Libraries 菜單,在 空位敲入 'alleg',然後將前面  
的選擇窗選上.   
 
- 如果你不想使用某部分 Allegro 函數,你可以 在包含 allegro.h 前定義下面的任何預處  
理標誌來稍微的減少你的一點編譯時間:   
 
 
alleg_mouse_unused  
alleg_timer_unused  
alleg_keyboard_unused  
alleg_joystick_unused  
alleg_gfx_driver_unused  
alleg_palette_unused  
alleg_graphics_unused  
alleg_vidmem_unused  
alleg_flic_unused  
alleg_sound_unused  
alleg_file_unused  
alleg_datafile_unused  
alleg_math_unused  
alleg_gui_unused  
 
int allegro_init();  
初始化 Allegro 庫. 它除了 設置一個全局變量,鎖住一些內存, 將 allegro_exit() 加載  
到 atexit() 例程 上以外沒有做更多的事,但是你必須在做其它事之前調用它. 成功者返回  
0 (它不可能調用失敗,所以 沒有更多的返回值).   
 
void allegro_exit();  
關閉 Allegro 系統. 這包括將系統返回到文本模式, 卸載裝載過的鼠標,鍵盤,時鐘例程.  
一般你不必去明顯的去調用它, 因為 allegro_init() 將它作為 atexit() 例程加載了, 所  
以它將在你的程序退出時自動被調用.   
 
extern char allegro_id[];  
一個包含有庫的日期和版本信息的字符串, 也許你在某些地方回用到這些.   
 
extern char allegro_error[];  
一個被 set_gfx_mode() 和 install_sound() 用來報告錯誤信息的字符串. 如果這些函數  
調用失敗而你想告訴用戶是為什麼,在這裡 能找到有關問題的描述.   
 
extern int os_type;  
由 allegro_init() 設置成以下值中的一個:   
 
      OSTYPE_UNKNOWN    - 未知, 或常規的 MSDOS  
      OSTYPE_WIN3       - Windows 3.1 或更早的版本  
      OSTYPE_WIN95      - Windows 95  
      OSTYPE_WINNT      - Windows NT  
      OSTYPE_OS2        - OS/2  
      OSTYPE_WARP       - OS/2 Warp 3  
      OSTYPE_DOSEMU     - Linux DOSEMU  
      OSTYPE_OPENDOS    - Caldera OpenDOS  
 
extern int windows_version, windows_sub_version;  
由 allegro_init() 設置, 使用 int 0x2F, ax=0x1600 來檢測 當前的 Microsoft  
Windows. 在 windows 3.1 下, windows_version 將被設為 3, 在 win95 下則是 4, 在 NT  
在將是 0x100, 如果沒有運行在 windows 下則為 0. 在 Linux DOSEMU 下, 兩個變量 都將  
被設為 -1.   
 
void check_cpu();  
檢測 CPU 型號, 設置以下的全局變量:   
 
extern char cpu_vendor[];  
如果可能,由 check_cpu() 設置成 CPU 廠商的名字.   
 
extern int cpu_family;  
由 check_cpu() 設置成 CPU 型號: 3=386, 4=486, 5=Pentium, 6=PPro, 等等.   
 
extern int cpu_model;  
由 check_cpu() 設置成 CPU 子類型. 如果是 386 (cpu_family=3), 0 表示 DX 芯片, 2  
表示 SX, 4 表示 SL, 15 表示未知. 如果是 486 (cpu_family=4), 0 或 1 表示 DX 芯片,  
2 表示 SX, 3 表示 487 (SX) 或 486 DX, 4 表示 SL, 5 表示 SX2, 7 表示 DX2  
write-back enhanced, 8 表示 DX4 或 DX4 overdrive, 14 表示 Cyrix, 15 表示未知. 如  
果是奔騰芯片 (cpu_family==5), 1 表示 Pentium (510\66,567\66), 2 表示 Pentium  
P54C, 3 表示 Pentium overdrive processor, 5 表示 Pentium overdrive for InterDX4,  
14 表示 Cyrix, 15 表示未知.   
 
extern int cpu_fpu;  
由 check_cpu() 設置成 TRUE 或 FALSE, 這個值依賴於有無一個浮點運算器 能夠被利用.   
 
extern int cpu_mmx;  
由 check_cpu() 設置成 TRUE 或 FALSE, 這個值依賴於是否有 MMX 指令可以被利用.   
 
extern int cpu_cpuid;  
由 check_cpu() 設置成 TRUE 或 FALSE, 這個值取決於 "cpuid" 指令 是否能被利用. (如  
果它被設置, 其它的 CPU 變量 100% 可信, 否則某些地方可能出了問題).
