============    調色板例程    ============ 
========================================== 
所有的 Allegro 繪圖函數使用整數參數來代表顏色. 
在真彩分辨率下這些數字以紅,綠,藍的聯合位來直接描述色彩, 
但是在常規的 256 色模式下 
這些值被看作是當前調色板的一個索引號, 這個調色板是 
描述 256 種可能的色彩中每一個顏色的紅,綠,藍的強度值的一個列表. 
 
調色板被儲存在一個 RGB 結構裡, 它包含了以 
VGA 硬件格式描述的紅,綠,藍的強度值,每一範圍是 0-63. 這個結構是 
這樣定義的: 
typedef struct RGB 
{ 
   unsigned char r, g, b; 
} RGB; 
例如: 
   RGB black = { 0,  0,  0  }; 
   RGB white = { 63, 63, 63 }; 
   RGB green = { 0,  63, 0  }; 
   RGB grey  = { 32, 32, 32 }; 
類型 PALETTE 被定義成 256 RGB 結構的數組. 
你將注意到 Allegro 裡的許多代碼將 'palette' 拼寫成 'pallete'. 
這是因為我在 Atari 上的舊的 Mark Williams 編譯的頭文件 
將它拼寫成兩個 l, 以至於我也這樣做. Allegro 很樂意 
接受其它的拼法, 只要在 allegro.h 裡加上一些 #define 就可以了. 
void vsync(); 
   等待一個垂直回掃的開始. 回掃發生在 
   你的顯示器裡的電子束到達屏幕的下方然後回到 
   上方準備下次掃描的時候.在這個短暫的時期, 
   圖形卡不向顯示器發送任何數據, 因此你能夠做一些 
   在其它時間不可能做的事, 比如改變 
   無閃爍(雪花)的改變調色板. Allegro 將自動 
   在改變調色板或做任何硬件滾屏之前等待回掃, 
   所以你一般不必去調用這個函數. 
 
void set_color(int index, RGB *p); 
   設置指定的 RGB 組到指定的調色板口. 不像 
   其它的調色板函數, 這個不去做任何的回掃同步, 所以你要 
   在調用它之前調用 vsync() 來避免雪花的問題. 
void _set_color(int index, RGB *p); 
   這是 set_color() 的在線(online) 版本, 為在 
   垂直回掃模擬器回叫函數里使用而設計. 它僅能在 
   VGA 13h 模式和 mode-X 下使用, 因為許多新的 SVGA 芯片 
   不是 VGA 兼容的 (set_color() 和 set_palette() 將在這些卡上使用 VESA 調用, 
   但是 _set_color() 不知道這些). 
void set_palette(PALETTE p); 
   設置整個調色板的 256 種顏色. 你應提供 256 個 RGB 結構的數組. 
   不像 set_color(), 沒有必要在這個函數之前調用 vsync(). 
 
void set_palette_range(PALETTE p, int from, int to, int vsync); 
   設置 from 到 to 之間的調色板口 (包括這兩個值:傳入 0 和 255 設置整個調色板). 
   如果 vsync 被設置則等待垂直回掃, 
   否則它馬上設置顏色. 
void get_color(int index, RGB *p); 
   取得指定的調色板口的值. 
void get_palette(PALETTE p); 
   取得整個調色板的 256 種顏色. 你應該提供 
   256 個 RGB 結構的數組,將數據存放在裡面. 
void get_palette_range(PALETTE p, int from, int to); 
   取得 from 到 to 之間的調色板口的值 (包括這兩個值:傳入 0 和 255 
   將得到整個調色板). 
void fade_interpolate(PALETTE source, dest, output, int pos, int from, to); 
   在 source 和 dest 之間計算出一個過渡調色板, 
   將它返回到 output 參數中. 在兩個極端間的位置 
   由 pos 變量指定: 0 則返回 source 的精確拷貝, 
   64 則返回 dest 的精確拷貝, 32 就返回兩個調色板的中間調色板, 等等. 
   這個例程只對 from 到 to 之間的顏色起作用 (包括這兩個: 
   傳入 0 和 255 兩處理整個調色板). 
void fade_from_range(PALETTE source, dest, int speed, int from, to); 
   從 source 調色板到 dest 調色板逐漸過度一部分調色板. 
   speed 從 1 ( 最慢) 到 64 (立刻變化). 這個 
   例程只對 from 到 to 之間的顏色起變化 (包括這兩個: 傳入 0 和 
   255 將過度整個調色板). 
void fade_in_range(PALETTE p, int speed, int from, to); 
   從黑屏到指定的調色板逐漸過度調色板. 
   speed 從 1 (最慢) 到 64 
   (立刻變化). 這個例程只對 from 到 to 之間的顏色起變化 
   (包括這兩個: 傳入 0 和 255 將過度整個調色板). 
void fade_out_range(int speed, int from, to); 
   將當前的調色板的一部分逐漸過度到黑屏. 
   speed 從 1 (最慢) 到 64 (立刻變化). 
   這個例程只對 from 到 to 之間的顏色起作用 (包括這兩個: 
   傳入 0 和 255 兩處理整個調色板). 
void fade_from(PALETTE source, PALETTE dest, int speed); 
   從 source 調色板逐漸過度到 dest 調色板. speed 
   從 1 (最慢) 到 64 (立刻變化). 
void fade_in(PALETTE p, int speed); 
   從黑屏逐漸過度到指定的調色板. speed 
   從 1 (最慢) 到 64 (立刻變化). 
void fade_out(int speed); 
   從當前的調色板逐漸過度到黑屏. speed 
   從 1 (最慢) 到 64 (立刻變化). 
void select_palette(PALLETE p); 
   暫時的變換,巧妙的使用在當你需要在調色板間和真彩圖像格式轉換形式下. 
   像 set_palette() 
   函數那樣設置內部調色板, 這樣轉換過程 
   將使用指定的調色板, 但是它卻不影響顯示硬件. 
   前一個調色板設置被保存在一個 
   內部緩衝裡, 可以調用 unselect_palette() 來恢復. 
void unselect_palette(); 
   恢復上次調用 select_palette() 調色板之前的調色板表. 
 
void generate_332_palette(PALETTE pal); 
   建立一個偽真彩調色板, 使用 3 位的紅色和綠色和 2 位的藍色. 
   如果文件沒有包含自己的調色板 load_bitmap() 函數將返回這個 
   (即, 你正在讀入一個真彩的位圖). 
 
int generate_optimized_palette(BITMAP *bmp, PALETTE pal, char rsvd[256]); 
   產生適當的減少指定的真彩圖像的 256 色調色板的版本. 
   rsvd 這個參數指向一個 
   像徵著那些顏色可以被使用的表: 零表示可以自由使用, 
   非零表示這個顏色被你自己保留. 如果 rsvd 是 NULL, 
   從映射將使用整個調色板. 
extern PALETTE black_palette; 
   這個調色板包含著一團黑色, 由過度(fade) 例程使用. 
extern PALETTE desktop_palette; 
   這個調色板由 Atari ST 低分辨率桌面使用. 我不好 
   確認這個為什麼還在這兒, 除了 grabber 和測試程序使用外,就沒什麼用途了. 
   這可能是 Allegro 裡 Atari 僅有的遺留代碼,  
   可能是不好意識把它去掉吧 :-)
