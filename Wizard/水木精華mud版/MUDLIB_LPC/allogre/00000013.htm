========== 塊復制(Blitting) 和精靈 =========== 
============================================== 
所有的這些例程都將被目標位圖的剪裁矩形所影響. 
 
void clear(bitmap); 
   將位圖全部清為顏色 0. 
void clear_to_color(BITMAP *bitmap, int color); 
   用指定的顏色清位圖. 
void blit(BITMAP *source, BITMAP *dest, int source_x, int source_y, 
          int dest_x, int dest_y, int width, int height); 
   從源位圖復制一個矩形區域到目標位圖. 
   參數 source_x 和 source_y 是將復制的源位圖上的部分的左上角, 
   dest_x 和 dest_y 是 
   目標位圖上的坐標. 這個例程將考慮 
   目標位圖的剪裁矩形, 如果你試圖復制一個源位圖外的區域同樣也將做剪裁處理. 
 
   你可以在任兩個位圖間復制, 甚至是兩個重疊的內存位圖 
   (即, 來源和目標是相同的, 或者一個是另一個的子位圖). 
   然而,你應當注意的是,許多 SVGA 卡不提供 
   分開的讀寫窗, 這就是說從屏幕的一個部分復制到另一部分時 
   需要使用一個在內存裡的臨時位圖, 
   這樣將極其的慢. 因此一個普遍的規則是 
   你應當避免在 SVGA 模式下的屏幕本身的復制. 
   另一方面,在 mode-X 下, 從屏幕的一個部分復制到另一部分 
   可以比從內存到屏幕的復制出奇的快, 
   只要源部分和目標部分正確的對齊. 
   在重疊的屏幕矩形間復制很慢,但是如果 
   區域不重疊, 而且它們有相同的面位置(plane alignment) (即: 
   (source_x%4) == (dest_x%4)), VGA 門鎖(latch)寄存器可以被用來做 
   很快的數據傳輸. 為了得到這個優勢, 在 mode-X 下經常值得在 
   顯存的隱藏區域儲存磚塊圖片(使用一個大的 
   虛擬屏幕), 然後再將它們復制到可顯的屏幕上. 
 
   不像所有的圖形例程, blit() 允許源位圖和目標 
   位圖有不同的顏色深度, 所以它可以被用來 
   在兩種點格式之間轉換. 
void masked_blit(BITMAP *source, BITMAP *dest, int source_x, int source_y, 
                  int dest_x, int dest_y, int width, int height); 
   像 blit() 那樣, 但是忽略透明點 ( 在 256 色模式下是 0, 
   真彩數據裡是亮點). 源圖片必須是內存位圖或它的子位圖, 
   並且源圖和目標圖區域絕對不能重疊. 
void stretch_blit(BITMAP *source, BITMAP *dest, 
                  int source_x, source_y, source_width, source_height, 
                  int dest_x, dest_y, dest_width, dest_height); 
   像 blit() 那樣, 不過它還可以調整圖片的比例因此源圖和目標圖 
   可以有不同的大小.這個例程沒有像常規的復制 
   那樣做常規的安全檢查: 你必須特別小心 
   不要復制源圖以外的區域, 而且你不能 
   在重疊區域間復制, 即你必須將源圖和目標圖使用不同的位圖. 
   同樣, 源圖必須是內存位圖或它的子位圖, 
   不能是硬件屏幕. 
void draw_sprite(BITMAP *bmp, BITMAP *sprite, int x, int y); 
   畫一個精靈位圖到目標位圖的 
   指定位置.這完全和 blit(sprite, bmp, 0, 0, x, 
   y, sprite->w, sprite->h) 相同, 但是它使用了掩式繪圖模式, 透明 
   點 ( 256 色下的 0, 真彩下的亮點) 被忽略, 
   因此背景圖片將透過精靈被顯示. 
   精靈必須是一個內存位圖, 而不是個屏幕位圖或其子位圖. 
   目標位圖可以是任何位圖. 
   然而它通常不支持混合顏色深度, 
   一個特例是這個函數可以將 256 色源圖畫到 
   真彩位圖上,因此你可以在真彩程序裡對指定的精靈使用調色板效果. 
 
void draw_sprite_v_flip(BITMAP *bmp, BITMAP *sprite, int x, int y); 
void draw_sprite_h_flip(BITMAP *bmp, BITMAP *sprite, int x, int y); 
void draw_sprite_vh_flip(BITMAP *bmp, BITMAP *sprite, int x, int y); 
   這些像 draw_sprite(),  但是它們翻折圖片,可以是縱向的, 
   橫向的, 對角線的軸變換. 這個產生了精確的鏡像圖片, 但它和 
   旋轉精靈不同 (它比旋轉例程要快的多). 
   精靈必須是內存位圖. 
void draw_trans_sprite(BITMAP *bmp, BITMAP *sprite, int x, int y); 
   使用一個全局色彩映射表或真彩色混合函數來將一個 
   精靈覆蓋在存在的圖片上面. 這個必須在你 
   已經設置了顏色映射表 ( 256 色模式) 或混合映射表 
   (真彩模式) 後才能使用. 因為它在寫圖時還牽扯了讀操作, 
   如果你直接向顯存畫半透明效果,將非常的慢, 
   因此無論何地有可能的話你都應當使用內存位圖來代替屏幕位圖. 
void draw_lit_sprite(BITMAP *bmp, BITMAP *sprite, int x, int y, int color); 
   使用全局色彩映射表向指定色彩或亮度級過度精靈圖片, 
   將算出的圖片畫到目標位圖上. 
   這個必須在你設置了顏色映射表 
   (256 色模式) 或 混合映射表(真彩模式) 後才能使用. 
void draw_character(BITMAP *bmp, BITMAP *sprite, int x, int y, int color); 
   將精靈位圖畫在位圖的指定 
   位置上, 將透明點 ( 256 色模式下的 0, 真彩模式下的亮點) 
   以當前的文本模式畫它們 (如果文字模式是 -1 則忽略它們, 
   否則以文本背景色畫它們), 
   並且將所有其它的點設置為指定的顏色.  
   精靈必須是 8 位圖片, 即使目標位圖是真彩的也不例外. 
void rotate_sprite(BITMAP *bmp, BITMAP *sprite, int x, int y, fixed angle); 
   旋轉位圖一個指定的角度,將它畫在位圖的指定位置. 
   角度是一個 16.16 的定點數. 這個定點數和 
   定點數三角例程中的一樣, 256 等於 
   整圈, 64 是直角, 等等. 精靈必須是內存位圖. 
 
void rotate_scaled_sprite(BITMAP *bmp, BITMAP *sprite, int x, int y, 
                          fixed angle, fixed scale); 
   像 rotate_sprite() 相同, 但是在旋轉的同時還做拉扯或壓縮的操作. 
 
void stretch_sprite(BITMAP *bmp, BITMAP *sprite, int x, int y, int w, int h); 
   將精靈圖片畫在位圖的指定位置, 
   拉扯它到指定的寬度和高度.  
   stretch_sprite() 和 stretch_blit() 的不同之處在於 stretch_sprite() 去掉 
   了透明點 ( 256 色模式下的 0, 真彩模式下的亮點).
