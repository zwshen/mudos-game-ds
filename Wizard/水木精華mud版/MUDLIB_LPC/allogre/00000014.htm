===================================== 
============   RLE 精靈  ============ 
===================================== 
因為位圖能夠以很多方式運用, 位圖的數據結構也就非常的複雜了, 
並且包含了許多的數據. 然而在許多情況下, 
你將發現你自己儲存的圖片僅僅只是需要復制到屏幕上, 
而不是用來做填充圖案等. 
如果是這樣, 你最好將你的圖片儲存為 RLE_SPRIT 
或 COMPILED_SPRITE 結構而不是常規位圖. 
RLE 精靈以一個簡單的運行行編碼格式儲存圖片, 在這種格式裡 
重複的零點用一個長度統計數來表示,  
非零點串由計數器給出長度. 
RLE 精靈通常比普通的位圖小的多, 即因為行壓縮, 
又因為避免了許多無用的位圖數據結構. 
它們經常比普通位圖要工作的快, 
因為它不用比較每個點是否是零來決定 
是否應該畫出來, 一個 add 就可能忽略掉整段零, 
而且可以用快速的串操作復制一大段非零的點. 
 
然而, 任何玉都有瑕疵, 由於 RLE 精靈 
缺少柔韌性.  你不能再在上面作畫, 不能翻折,旋轉,或拉伸 
它們. 事實上你唯一能做的是 
使用 draw_rle_sprite() 函數將它畫在位圖上,  
就相當於使用 draw_sprite() 畫一個常規的位圖. 你可以在 
運行時將位圖轉換成 RLE 精靈, 你也可以在 grabber 數據文件中 
選 'RLE' 按紐建立位圖來創建 RLE 精靈. 
RLE_SPRITE *get_rle_sprite(BITMAP *bitmap); 
   以指定的位圖為基礎來創建一個 RLE 精靈 (必須是 
   內存位圖). 
void destroy_rle_sprite(RLE_SPRITE *sprite); 
   摧毀一個由先前 get_rle_sprite() 創建的 RLE 精靈結構. 
void draw_rle_sprite(BITMAP *bmp, RLE_SPRITE *sprite, int x, int y); 
   將一個 RLE 精靈畫在位圖的指定位置. 
void draw_trans_rle_sprite(BITMAP *bmp, RLE_SPRITE *sprite, int x, int y); 
   draw_rle_sprite() 的半透明版本. 描述見 
   draw_trans_sprite(). 這個只能使用在你設置了 
   色彩映射表 ( 256 色模式) 或混合表 (真彩模式) 以後. 
 
void draw_lit_rle_sprite(BITMAP *bmp, RLE_SPRITE *sprite, int x, y, color); 
   draw_rle_sprite() 的淡化(Tint) 版本. 描述見 
   draw_lit_sprite(). 這個只能使用在你設置了 
   色彩映射表 ( 256 色模式) 或混合表 (真彩模式) 以後.
