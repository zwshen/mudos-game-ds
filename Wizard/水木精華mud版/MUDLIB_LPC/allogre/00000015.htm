============  編譯(Compiled)精靈 ========= 
========================================== 
編譯精靈是以現行機器碼指令形式儲存的, 可以將 
指定的圖片畫在位圖上, 它使用了 mov 指令和及時數據變量. 
這是畫掩式圖片的最快方法: 在我的機器上 
畫被編譯過的精靈大約比使用 draw_sprite()  
畫一個常規位圖快 5 倍. 編譯精靈比較大, 所以如果內存比較緊張的話你 
應該使用 RLE 精靈來替代它, 並且它比 RLE 精靈有更多的限制, 
因為他們不支持剪裁. 如果 
你試著向位圖邊緣畫, 將使內存操作不可靠而且極有可能使系統死掉. 
你可以在運行時將位圖轉換成編譯精靈, 
或者你可以在 grabber 數據文件裡, 建立一個位圖對像然後 
點選 '線形編譯' 或 ' Mode-X 編譯' 按紐來創建編譯精靈 
 
COMPILED_SPRITE *get_compiled_sprite(BITMAP *bitmap, int planar); 
   以指定位圖來創建一個編譯精靈(必須是內存位圖). 
   編譯精靈是設備關聯的, 因此你必須 
   指定是編譯成線形的還是平面的格式. 如果你將畫在 
   內存位圖或 mode 13h 和 SVGA 屏幕位圖上, 第二個參數應傳入 FALSE, 
   如果你將畫在 mode-X 或 Xtended 模式屏幕位圖上, 就傳入 TRUE. 
 
void destroy_compiled_sprite(COMPILED_SPRITE *sprite); 
   摧毀一個由前面 get_compiled_sprite() 得到的編譯精靈結構. 
 
void draw_compiled_sprite(BITMAP *bmp, COMPILED_SPRITE *sprite, int x, int y); 
   向位圖的指定位置畫一個編譯精靈. 
   精靈必須被編譯成正確的位圖類型 (線形或平面). 
   這個函數不支持剪裁. 
   建議: 也許不能剪裁編譯精靈並不是個問題, 一個小技巧 
   是設置工作平面(內存位圖, mode-X 虛擬屏幕 或其他) 
   比你實際需要的大那麼一點點, 然後使用中間的一部分. 
   這樣你可以稍稍的超出邊界而沒什麼麻煩了...
