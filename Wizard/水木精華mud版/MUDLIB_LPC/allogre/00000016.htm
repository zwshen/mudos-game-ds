=====================================  
============   文字輸出  ============  
=====================================  
Allegro 提供了文字輸出例程, 可以使用比例字體(單個字符可以是任何  
大小) 和固定大小為 8x8 或 8x16 字體.字體包含了 ASCII 字符 32 到  
255: 所有其它的字符將被空格代替. grabber 程序能從一個 PCX 文件上  
畫著的字符集來建立字體(見 grabber.txt 獲得更多信息),也可以從 GRX  
或 BIOS 字體格式文件獲取.  
 
extern FONT *font;  
   一個簡單的 8x8 固定大小字體 (mode 13h BIOS 缺省). 如果你想改  
   變供 GUI 例程使用的字體, 將這個指針改向你自己的字體.  
 
void text_mode(int mode);  
   設置文字畫向位圖的模式. 如果 mode 為 0 或正,文字輸出將不通明  
   且字符的背景將被設置成顏色 #mode. 如果 mode 是負數, 文字將以  
   透明方式畫在位圖上 (即: 字符的背景將不會改變). 缺省值為零.  
 
void textout(BITMAP *bmp, FONT *f, unsigned char *s, int x, y, int color);  
   寫一個字符串 s 到位圖的 x,y 位置, 使用當前的文字模式指定的字  
   體和前景顏色. 如果 color 是 -1 且正在使用比例字體, 它將使用  
   字體位圖的原始顏色(你從 grabber 程序讀出的) 這允許了多色彩的  
   文字輸出.  
 
void textout_centre(BITMAP *bmp, FONT *f, unsigned char *s, int x, y, color);  
   和 textout() 相似, 但是將 x 坐標解釋為字符串的中間而不是它的  
   左邊界.  
 
void textout_justify(BITMAP *bmp, FONT *f, unsigned char *s,   
                     int x1, int x2, int y, int diff, int color);  
   在區域 x1-x2 之間畫上對齊的文字. 如果空餘空間的數量比 diff 值  
   要大, 則將被放棄並用常規的左對齊代替.  
 
void textprintf(BITMAP *bmp, FONT *f, int x, y, color, char *fmt, ...);  
   格式化文字輸出, 使用 printf() 形式的字符串.  
 
void textprintf_centre(BITMAP *bmp, FONT *f, int x, y, color, char *fmt, ...);  
   和 textprintf() 相似, 但是將 x 坐標解釋為字符串中央而不是左  
   邊界.  
 
int text_length(FONT *f, unsigned char *str);  
   返回以指定字體構成的字符串的長度(點為單位).  
 
int text_height(FONT *f)  
   返回指定字體的高度 (點為單位).  
 
void destroy_font(FONT *f);  
   釋放正在被字體結構使用的內存.
