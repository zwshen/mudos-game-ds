標  題: 關於這個版本的mudos的使用 
發信站: BBS 水木清華站 (Wed Aug 12 22:55:33 1998) 
 
 
    我編譯的這個v22版本mudos和它原來的v21.7的mudos用法基本一致，都是 
通過一個mudos.ini文件來傳遞參數的。 
 
    Mudos執行程序是mudos.exe文件。使用它之前要在mudos.exe相同目錄下， 
新建一個文本文件，取名為mudos.ini。內容如下 
 
; mudos.ini 
[init] 
config=d:\mudlib\es2\config.es2  ; 這個是mudlib的配置文件路徑，最好用全路徑名 
heartbeat=1000                   ; 這個是heartbeat的值，以毫秒計 
addr_port=3990                   ; 如果需要啟動addr_server，請加上此項。 
 
[com] 
{EAB22AC1-30C1-11CF-A7EB-0000C05BAE0B}=c:\win\system\shdocvw.dll 
 
 
   v22版的mudos在使用com之前，mudos本身要做一次初始化操作。請運行 
   mudos register 
   如果在NT上跑的話，要用管理員登錄，運行mudos install來添加服務 
 
   啟動mudos時，95下只用運行mudos就可以。NT下就要在Control Panel中 
的服務一項，啟動mudos v22pre11 ...那個服務(這個服務僅當你運行了 
mudos install之後才會出現) 
 
   mudos啟動後，不論是95還是nt，在那個Explorer的任務條的右端都會 
出現一個小地球的圖標。鼠標右鍵點擊他，會出現一個菜單讓你選擇是 
關閉還是重起。 
 
 
   V22版在網上同時發佈了源碼。請把這些源碼展到一個叫mudos的目錄下 
然後將上層目錄映射成x: 和 z: 
   如你將它們展到了 c:\mud\mudos下，就該運行 
      subst x: c:\mud 
      subst z: c:\mud 
   然後運行VC5，打開x:\mudos下的那個工程文件，先編譯editsource工程 
再編譯mudos工程就可以了。 
   編譯時需要一個yacc工具，在 
   ftp://ftp.lib.pku.edu.cn/incoming/MudosV22pre11/ 
   下可找到，是bison.exe bison.hai bison.sim三個文件 
   (注意這個工具和網上常見的有不一樣的地方，是專門為編譯 
這個mudos編譯出來的yacc工具)
