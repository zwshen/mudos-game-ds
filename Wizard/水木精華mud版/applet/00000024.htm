mudos新版本（新到什么程度就不知道了）提供binary的模式。（一般 
是telnet的模式，其实也差不多）。在config.xxx里写： 
external_port_1 : telnet 5000 
external_port_2 : binary 5004 
等等，这样5004的端口就是binary的啦，可以用来做各种服务， 
master.c中的connect的那个端口的参数就有作用了，根据不同的端口 
启不同的object，这个object用process_input处理收到的东西，可以用 
printf, receive等来返回给客户端。这样可以写些很简单的服务，比如http :) 
SMTP也可以，不过一般的端口占不到:PP 
 
所以改该sendmail什么的，是不是可以给mud里的玩家一个id.shell@some.mud.com 
的email :PP ( 比如nerd.mud@xo.... )
