shutdown(3)　 MudOS v21c2　 (5 Sep 1994)

　名稱:
    　 shutdown() - 關閉 mud 驅動程序 (driver).
　語法:
    　 void shutdown( int how );

    　 無返回值 shutdown( 整數 how );

　用法:
    　 這個函數可以關閉驅動程序. 比起因為程序錯誤、文件損毀、驅動程序被 kill 或其他原因導致驅動程序發生嚴重問題而關閉, 用 shutdown 算是正常控制驅動程序關閉的方式. 參數 how 指定驅動程序要傳遞給 exit() 函數的整數值. 慣例上, 當 how 為 -1 時, 則一起關閉重新啟動驅動程序的 script . 所以 reboot (重新啟動 mud) 這個指令應該使用 shutdown(), 而 halt 指令 (停止 mud 執行) 應該使用 shutdown(-1) .　 如果您想要藉由上述的方式控制 mud 關閉的方式, 您應該在 script 中檢查返回值.
    　 shutdown() 顯然是一個需要安全把關的函數. 在 simul_efun 中加上 shutdown()　 並使用 exec(), 這樣可以對執行 shutdown() 作適當的檢查. 請確定在主控物件 (master.c) 中也要設定 valid_override(4) 以防止 efun::shutdown() 這種方式逃過檢查.
　參考:
    　 crash(4), slow_shutdown(4)
