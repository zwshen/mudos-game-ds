get_char(3)　 MudOS v21c2　 (5 Sep 1994)

　名稱:
    　 get_char - 讓之後輸入的字符送往一個指定的函數.
　語法:
    　 varargs void get_char( string | function fun, int flag, ... );

    　 不定參數 無返回值 get_char( 字符串 或 函數指針 fun, 整數 flag, ... );

　用法:
    　 讓使用者之後輸入的字符送往函數指針 fun, 作為 fun　 的參數. 驅動程序 (driver) 不會分析 (parse)　 輸入的字符.

    　 請注意 get_char 是「非阻隔性」 (non-blocking) 的, 意即調用 get_char 的物件不會暫停下來等待使用者的輸入動作. 調用 get_char 的物件會繼續執行隨後任何的程序語句. 在收到輸入的字符之前, 指定的函數 fun　 不會被調用.

    　 如果同一個物件調用 get_char() 超過一次以上, 就只有第一次的調用有效.

    　 如果可以選擇的參數 flag 指定為非零值 (non-zero),　 則玩家輸入的字符就不會回傳 (echo) , 物件被竊聽時也不會回傳 (用在輸入密碼時很有用).

    　 調用函數指針 fun　 時, 使用者輸入的字符會被當作是函數的第一個參數 (變量類型為字符串) . 在 flag 之後指定的參數會當作函數 fun　 的其他參數.

　問題:
    　 get_char 在 MudOS 0.9 及更早的版本有一個非常明顯的問題. 在較差之遠端終端機協議 (telnet negotiation) 的許多作業系統上 (按: 所有市場上早期的主流品牌之工作站) , get_char　 會使畫面輸出的結果變得非常奇怪. 除非這些問題得到解決, 在一般的 mudlib 源代碼中, 還是建議別用 get_char　 這個函數.　 (目前只知道 get_char 對使用 NeXT 工作站連線的使用者不會有此問題. )
