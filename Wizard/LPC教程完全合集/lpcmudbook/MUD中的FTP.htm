MUD中的FTP
 2002-8-13  


　　MUD中的FTP FTP ( File Transfer Protocol ) 是如今在Internet上广泛使用的一种用于文件传送的协定。我们可以通过它在远程电脑upload和download文件。当然MUD也不能例外,大量的自制文件必须通过FTP传送到服务器下。
　　其实现在MUD里面用FTP真的很简单，现有供下载的MUDLIB中都有完整的FTP Server程序。你只需要改改配置就可以用了。
　　言归正传，MUD中的FTP主要包含以下几个文件：
　　 /include/net/ftpdsupp.h
　　 FTPD的支持文件 其中主要有检查password和site的code。

　　 /include/net/ftpdconf.h
　　 FTP的配置文件（只是想要架站的朋友看这里就行了）

　　 #define FTPD_PORT
　　 即登录FTP的Port，只要设置成和进入MUD的Port不同就行了。

　　 #define FTPD_TIMEOUT
　　 在FTP上所能发呆的default值，按秒计算。超过这个时间就会自动断线。

　　 #define MAX_FTPD_IDLE
　　 在FTP上所能发呆的最大值，和上面那个不同的是可以由siteidle指令人工设定。

　　 #define HOME_DIR(name) 使用者数据来自的目录。

　　 #define LOGIN_MSG
　　 进入FTP时的欢迎图像，一般应存于/adm/etc。

　　 #define THE_MUD_NAME 你的MUD的名字啦。

　　 #define FTPD_MAX_USERS 最多能同时使用FTP的人数。

　　 #define THE_VERSION Mudos的版本。不是ftp文件的版本喔。

　　 #define BLOCK_SIZE 传送文件的什麽中断值，不管它啦。

　　 #define FTP_BUGS_EMAIL
　　 如果FTP出现问题时需要联系的Email，如果你有改动就填你的Email吧。

　　 #define FTP_USERS 允许使用FTP的用户名称。

　　 #define LOG_FILE FTP记录文件。

　　 下面一些都是杂七杂八的Define，不是很重要，我就随便讲讲了。

　　 ANONYMOUS_FTP ： 没有id的用户登录FTP。
　　 GUEST_WIZARD_FTP ： 没有home目录，即/u下目录的巫师登录FTP。
　　 FILE_LOCKING ： 对特殊的文件进行保护。
　　 CHECK_SITE ： 对个别ID（如admin级）进行site限制。
　　 MESSAGE_FILES ： 当转换路径时给出的信息。
　　 DEBUG ： 除错功能

　　 本来还有一些啦，我觉得没什麽用就懒得写，如果不懂可以post。

　　 /include/net/ftpd.h
　　 FTPD的头文件 在FTPD里面所要用到的define都在这里。包括读写权限等等。

　　 /adm/daemons/ftpd.c
　　 这就是FTP的主文件啦 相当规范的Code，光看前面的更新记录就很有帮助喔。

　　 int check_valid_read() 检查读入权限，权限参照ftpdconf.h。

　　 int check_valid_write() 检查写入权限。

　　 mapping *query_connections() 当前FTP的联接状况

　　 void setup_ftp() 建立FTP Socket。

　　 void in_listen_callback() & void in_write_callback()
　　 处理一些在connection中出现的问题，如人数过多，timeout等等。

　　 string ls() 在FTP中的MUD文件目录显示。

　　 data_write_callback() FTP的数据传入（transfer）。

　　 static void data_conn() & static void read_connection()
　　 在FTP中的连接以及读出数据。

　　 data_close_callback() 数据关闭，一些写入权限也在这里限制。

　　 void logout() 退出FTP。

　　 void parse_comm() 连入FTP，在FTP中使用的各种指令，都是在这里实现的。

　　 void check_connections() 检查FTP的连接状态。

　　 void resolve_callback() 解决一些在FTP使用中出现的问题。

　　 这里讲一个小应用，如何监测FTP，可以即时显示FTP的使用情况。
　　 先在CHANNELD中设置一个新的FTP频道，放在现有的系统频道也可以。再在FTPD中 void create()中set channel_id，然后用CHANNEL_D->do_channel在相关event的code中送出FTP用户使用的情况。
　　 如：退出FTP就在void logout()中加入： CHANNEL_D->do_channel( this_object(), \"ftp\", sprintf(\"%s 退出FTP。\", UNAME )); 这样就送出了即时信息。
　　 相同的意思，log in和command的信息在void parse_comm()加入就行了。如果你觉得太多太Flood 可以只选一部分进行监测，如login，logout，delete，upload等较重要的信息。这样的话可以即时处理一些紧急情况，保证 MUD更安全的运行。
